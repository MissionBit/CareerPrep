Forms & Positioning

Collect user input with forms!

HTML forms are used to collect user input. The <form> element defines an HTML form. Form elements are different types of input elements, checkboxes, radio buttons, submit buttons, and more. Try it out on your page:

<form>
  <input type="text">
  <input type="password">
  <input type="email">
  <input type="submit">
</form>

Great! let's try a more complex example.

Write it out!! Don't use copy & paste!

Thinking about network traffic

Exercise: Take five minutes to understand How the internet works.

  • A GET request asks for information (like an HTML document, CSS file, or image) from the server
  • A POST request sends information (like a user's form input or a file to upload) to the server

We can also use HTML attributes to change the behavior of form elements or do form validation to only allow users to submit the form when certain conditions are met. Try out a few:

<input type="text" name="country_code" pattern="[A-Za-z]{3}">
  • disabled -- Specifies that an input field should be disabled
  • max -- Specifies the maximum value for an input field
  • maxlength -- Specifies the maximum number of character for an input field
  • min -- Specifies the minimum value for an input field
  • pattern -- Specifies a regular expression to check the input value against
  • readonly -- Specifies that an input field is read only (cannot be changed)
  • required -- Specifies that an input field is required (must be filled out)
  • value -- Specifies the default value for an input field

Let's give it a go!

Exercise 1: A Page for Hipsters

Hipsters have money and buy ridiculous things. Let's take advantage of this situation by building a very hip 2-column website to sell them hand-stitched leather whatevers.

Step 1: no layout

Create a new HTML document with a bunch of text from hipster ipsum. We want a lot of text to make them feel at home. Have at least 10 paragraphs.

Having no layout makes block elements (like <div>,<h1>,<p>) really wide, which gets kind of annoying to read.

Step 2: margin auto

#container {
  width: 960px;
  margin: 0 auto;
}

Let's add a container div to surround the content on our page with the above styles applied. We've seen this before. It just gives a fixed width to the content and centers it.

Step 3: flexible sizing for small screens

Try squishing the browser window really narrow though. What happens?

#container {
  max-width: 960px;
  margin: 0 auto;
}

What if we change our CSS to use max-width instead. What changes?

Step 3: brand and navigation divs

  1. Let's put our content inside a div#content (still inside #container)
  2. Above div#content (still inside #container), add a div#header, which should contain div#brand and div#nav
  3. Give every id'd div a different background-color so we can see what's going on.
  4. Place a logo and company name in #brand
  5. Mock out a few fake links for the #nav
  6. The navigation links should be an unordered list.
  7. How can we make the list items horizontal? Try it first, then check the solution

Step 4: fixed header

What if we want our branding and navigation to be visible no matter how far the user scrolls? Let's tell the browser we want the element with the id "header" to stay in a fixed position no matter how we scroll the screen, as if it were a sticker, stuck on the user's screen.

#header {
  position: fixed;
  top: 0;
  left: 0;
}

Discover the following:

  1. We've taken the #header out of the page flow. How do we prevent it from overlapping our content?
  2. What if we place our #header above #container instead of inside it?
  3. What if we give the header a set height and width?
  4. What if we give the the "top" and "left" properties non-zero pixel values (like 10px)?
  5. What if we change the "top" and "left" properties to "bottom" and "right" respectively?

Step 5: a second column

Okay things are getting serious now...

  1. Create a div#sidebar as the last element inside the #container
  2. Put a bunch of pictures of snazzy hipster things to buy in there
  3. Give #container a css rule position:relative; why?
  4. Give #content css rules position: absolute; width: 600px; why?
  5. Give #sidebar css rules width: 350px; margin-left: 600px; why?
  6. Confused yet? Check out this quick demo illustrating the different types of positioning

Okay, the above works, but it's a bit confusing, it doesn't squish itself for small screens, and for complicated reasons, if our position: absolute; element is larger than our container, it overflows its bounds.

Step 6: A better way

Another CSS property which has been hacked into layout duty is float. Float was originally intended for wrapping text around images. Try adding a few images mixed in with the hipster ipsum inside any paragraph tags just mixed into the middle of the text. Observe the result, and then give them the following style:

#content img {
  float: right;
  margin: 0 0 1em 1em;
}

Now try floating them left instead.

Okay, now for the big finale!

  1. Remove the position, and margin-left css properties from #container, #content, and #sidebar respectively
  2. Add float:left to both #content and #sidebar
  3. Where did the background color go from our #container?
  4. Add overflow: auto; to #container
  5. Change the widths of #content and #sidebar to 75% and 25% resepectively and try making the browser window really narrow.
  6. What is the difference in behavior between #container having 'width' vs 'max-width' when making the browser narrower?

Step N: clean-up

Let's add some padding, margins, borders, background colors, etc to make things look nice. What happens if we don't use the border-box CSS property on every element?

* {
  box-sizing: border-box;
}
