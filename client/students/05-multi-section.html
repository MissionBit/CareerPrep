Multi-Section Web Pages

Ok, now that we have our styling in order, let's think about how to build larger web pages in an orderly way...

<div id="greeting" class="section">
  <h1>Hello!</h1>
  <p>This content is all related!</p>
  <p>Nice that we can group it using a "div" tag!</p>
</div>

The <div> element is a generic block element which defines a section of your page. Unlike block elements with specific meaning and implied default styling (like <h1> and <p> tags), <div> elements don't change the appearance of the page until styles are applied to them.

<p>Sometimes, <span class="highlight">all we really want</span> is
to be able to refer to certain sections of our page, but we want to completely
<span class="highlight">control their styling</span>, and not get the
"b" or "i" tags involved</p>

The <span> element is a generic inline element which defines a section of your page. Unlike inline elements with specific meaning and implied default styling (like <b> and <i> tags), <span> elements don't change the appearance of the page until styles are applied to them.

Exercise: My First Blog Post

You know what your community site needs? Some content to attract members!

  1. Create a blog post detailing what a new member to your community might need to know to get up to speed with the group.
  2. The blog post should have- each in its own <div>- a header area, a nav bar area, a content area, and a footer area (with some contact info, disclaimer, etc).
  3. The content area should have multiple header and paragraph tags. Add some styles for each tag type.
  4. Give each <div> its own id attribute ("header", "nav", "content", and "footer"), and style each a bit (maybe they each have a different background color).
  5. Add some generic <span> tags around bits of important text. Give each a class of "important", and style them so that information stands out! Maybe a yellow background, red text, a heavier font-weight and a slightly larger font-size
  6. Add some <span> tags around bits of un-important text. Give each a class of "nice-to-know", and style that text a bit lighter and smaller.

The CSS Box Model

Every HTML element has a box around it, even if you cannot see it. We can show a visible border around every HTML element on the page using the CSS border property and the "match all" selector:

* {
  border: 1px solid red;
}

Exercise: Recreate the "Cottage Garden" page above, red- & green-boxes included!

The CSS padding property defines the extra space inside the border:

div {
    padding: 10px;
}

The CSS margin property defines the extra space outside the border:

div {
    margin: 30px;
}

We can also manually set the width and height of the element itself. When you do this, you'll want to set the box-sizing for obscure reasons:

div {
    box-sizing: border-box;
    width: 300px;
    height: 200px;
}

What's the difference? Try giving the selected elements a different background color than the rest of the page... Also, use the Elements panel of the Chrome Dev Tools [CMD + OPTION + I] to inspect the spacing around the element.

Exercise: Boxes, Boxes, Everywhere!

Make a very boxy web page...

  1. Create a bunch of dang divs and spans on your page.
  2. This is a web page about boxes. Add content about boxes. Cardboard boxes, metal boxes, self-imposed emotional boxes, whatever...
  3. Using a mix of class, id, and tag selectors, give boxes different background-colors.
  4. Give boxes a set height, width, border, padding, and margin.
  5. Answer the following questions:
    1. Can images have borders, margins, and padding?
    2. What happens when you give an element so much padding that it's larger than the height and width screen
    3. What's so great about boxes anyway?
  6. Figure out what "giving" an element each of these css rules does:
    1. border-radius: 10px;
    2. margin-top: 20px;
    3. padding-right: 20px;
    4. border-left: 10px solid red;
    5. padding: 20px 10px;
    6. padding: 20px 5px 10px;
    7. padding: 20px 5px 10px 50px;
  7. Try this on a div which wraps all other elements of the page... width:960px; margin: 0 auto; What does this do? Why would we want it?

Exercise: I need space!

You will be judged by professional developers for unorganized HTML code... Look back over the HTML documents you've created up to this point. For each one:

  1. Wrap all of the HTML elements in the body in a div with an id of "container". Give it the following css rules: width:960px; margin: 0 auto;
  2. Identify distinct sections, and wrap them in <div>'s
  3. Give those divs ids or classes to identify them
  4. Give divs some margin, padding, and borders as appropriate.
  5. Find places where you've used <b> and <i> tags and replace some of them with styled <span>'s
  6. Give some elements borders
