Exercise: Choose your own adventure

Create a series of html pages for a text-based "choose your own adventure" story, so that users can click different links to make choices to advance the story as they see fit. If you'd rather, you can create a "gift chooser" or "decision maker" feature for your community site. (i.e. "Which country should I travel to next?" for a travel-related group)

It would be a drag to have to rewrite that CSS code on each page... Let's use a link tag to share a single CSS document between multiple HTML files:

<head>
  <link href="styles.css" rel="stylesheet">
</head>
  1. Create a "template" first with the style and structure that you'll want common to all pages and create a copy to work from for each individual page
  2. Each page should have multiple links to choose the next page, and perhaps sometimes a link to jump back to the last page or even to the beginning
  3. Use some div and span elements
  4. Use some class and id attributes
  5. Use some padding, margin, and border css properties
  6. Use some complex css selectors
  7. Use some css rules that are more specific and thus overwrite other more general rules for a particular element

Exercise: A second Blog Post!

Your first post was such a roaring (fake) success, we should do another! We'd like to keep all the same styles consitent with the last post, but re-writing all that CSS would sure be a drag...

  1. Create a (short) new blog post about a upcoming or current event of interest to the community.
  2. Use a <link> tag to share a single .css file between both blog post HTML documents.
  3. Now take the css rules from inside your <style> and place them into the .css document.
  4. Link your blog posts together with each other and the rest of the site using a navigation bar
  5. Expand your linked styles.css to share the all styles site-wide. Replace your <style> tags on all community site pages with <link> tags pointing to styles.css
